How Permissions Work in Power Pages

Permissions in this portal are enforced at two layers:

  1. Power Pages Web Roles — Map Azure AD security groups to Power Pages web roles. Each role gets a different table filter on the Dataverse folder/table permissions.
  2. Dataverse Row-Level Security — Use the Access Team or ownership-based RLS so users only see rows they're permitted to access, even if the UI changes.
  3. Power Automate Approval — Approval flows check the requester's manager from Azure AD and route accordingly. HR forms route to the HR group.

Handoff Guide

How to add new forms and change user permissions in this Power Pages solution.

Adding a New Form

1
Create a Dataverse Table
In your Power Platform environment, create a new table (e.g. pnp_TravelRequest). Add columns for each form field. Include a Status choice column (Draft, Pending, Approved, Rejected) and a pnp_Requester lookup to the User table.
2
Add a Power Pages Form
In the Power Pages Studio, go to Data > Forms, click Add Form, select your new table, and choose "Create new form." Place it on a new or existing webpage. Set the form mode to "Create" for submissions.
3
Configure the Approval Flow
In Power Automate, create a new flow triggered by "When a row is added" on your Dataverse table. Add an Approval action assigned to the requester's manager (use "Get manager (V2)" connector). On approval/rejection, update the Dataverse row status and send an email notification.
4
Add Navigation & Permissions
Add a link in the Power Pages sidebar (or in this prototype's navigateTo() function). Then go to Security > Web Roles, edit the appropriate roles, and under Table Permissions, grant Create/Read access to the new table for the right roles.

Changing User Permissions

1
Map Azure AD Groups to Web Roles
Go to Security > Web Roles. Each web role (e.g. "Employee", "Manager", "HR Admin") should be linked to an Azure AD security group via the Identity Providers configuration. To change a user's role, add/remove them from the Azure AD group — changes sync automatically.
2
Adjust Table Permissions per Role
Under each web role, click Table Permissions. For document folders, use Filter rows to restrict which rows a role can see — e.g. filter by pnp_FolderAccessRole column. This is how employees only see their department's folders while HR sees all.
3
Control Page Visibility
In Power Pages Studio, select a webpage, go to Properties > Permissions, and restrict which web roles can view it. Pages like "Approvals" should only be visible to Managers and HR; "Permissions" only to HR. This is the equivalent of the style="display:none" logic in this prototype.
